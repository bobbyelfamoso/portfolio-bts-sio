# [C14] - Planification des activités

## Compétences mises en œuvre
*   **C14.1.1** : Définir les étapes d'un projet.
*   **C14.1.2** : Estimer les charges et les délais.
*   **C14.1.3** : Utiliser un outil de gestion de projet collaboratif.

## Contexte
Le hackathon s'est déroulé sur 48 heures. Sans une planification stricte, le projet FlashMind n'aurait pu être livré. Nous avons adopté une méthodologie **Agile (Scrum)** pour découper le travail.

## Démarche suivie
1.  **Répartition des Sprints** : Utilisation de **Trello** avec trois colonnes principales :
    *   **Backlog** : Idées et tâches initiales (Logo, Firebase config, GLPI setup).
    *   **Sprint** : Tâches prioritaires en cours (Rédaction CDC, Inventaire).
    *   **Terminé** : Tâches validées et testées.
2.  **Coordination de l'équipe** : Réunions régulières sur **Discord** pour lever les points de blocage ("Daily stand-up" accéléré).
3.  **Suivi de version** : Utilisation de **GitHub** pour fusionner les développements du front-end et du back-end (Gemini API) sans perte de code.

## ✅ Résumé des points d’attention
> [!NOTE]
> La planification via Trello a permis au Chef de Projet de visualiser instantanément la charge de travail de chaque membre et de réaffecter des ressources si nécessaire.
